# 실하드웨어 QA 체크리스트 (설계 T10)

출시 전 **실 Windows 하드웨어(x64 미니PC + 멀티모니터)**에서 최종 QA. 개발은 단일 모니터·100% DPI
환경에서 했고, **입력 합성이 차단**돼 라이브 상호작용은 자동 검증하지 못했다. VM/개발환경이 가리는 것:
GPU 투명 합성·Per-Monitor DPI·멀티모니터 좌표 정합·실부하 후킹·세션 전환·화면 녹화 캡처.

표기: **[auto]** = 개발 중 단위 테스트/셀프테스트로 검증됨(렌더 데이터·순수 로직). 그래도 실HW에서 실제
화면/상호작용은 재확인. 그 외 항목은 **이 체크리스트가 첫 실증**.

현재 자동 테스트: Core 285 + App 67 = **352 green**.

---

## 1. 환경 매트릭스

각 조합에서 핵심 항목(특히 §M·§N·§O)을 반복:

- [ ] **CPU/OS**: x64 미니PC, Windows 11 (그리고 가능하면 Windows 10 22H2)
- [ ] (선택) ARM64 기기 — `win-arm64` 빌드
- [ ] **모니터 수**: 단일 / 듀얼 / 트리플
- [ ] **DPI**: 100% / 150% / 200%, 그리고 **혼합 DPI**(예: 주 150% + 보조 100%)
- [ ] **배치**: 가로 나란히 / 상하 / 음수 좌표(주 모니터 왼쪽에 보조)
- [ ] **GPU**: 내장(Intel/AMD) + 외장(있으면). 원격 데스크톱(RDP) 세션도 1회

## 2. 사전 준비

- [ ] `pwsh scripts/publish.ps1 -Rid win-x64` → self-contained 단일 exe 생성
- [ ] (서명 시) `scripts/sign.ps1`로 서명 후 SmartScreen 동작 확인
- [ ] 대상 기기에 복사 후 실행(무설치). **첫 실행 시 수초 압축해제** 정상인지

## A. 설치·기동·종료
- [ ] 더블클릭 실행 → 트레이 아이콘(사이안 링) 표시, 작업표시줄 미노출
- [ ] exe 파일 아이콘이 Cluxo 링 글리프 **[auto: 임베드 확인]**
- [ ] 트레이 우클릭 → 메뉴(그리기/좌표/설정.../종료) 표시
- [ ] **종료** 클릭 → 트레이 아이콘 사라지고 프로세스 종료, 잔여 프로세스 없음
- [ ] 종료 후 재실행 시 설정 유지 **[auto: 직렬화]**

## B. 오버레이 렌더 기본 (★ GPU 투명 합성 — VM이 가림)
- [ ] 오버레이가 **투명**(뒤 데스크톱/앱이 그대로 비침), 불투명 박스 없음
- [ ] **클릭 통과**: 평소 오버레이 위 클릭이 밑 앱으로 전달됨
- [ ] **항상 위**: 다른 창 위에 효과가 보임(전체화면 앱 포함 — 게임/PPT 슬라이드쇼)
- [ ] 잔상/티어링/깜빡임 없음

## C. 커서 강조 (링)
- [ ] 커서 따라 링이 부드럽게 이동 **[auto: 위치 폴링]**
- [ ] 설정에서 **색/크기/투명도** 변경 → 즉시 반영 **[auto: RingVisual]**
- [ ] **모양**: 원/둥근사각/마름모/육각 전환 **[auto: 렌더 4종]**
- [ ] **외곽선 두께/점선** 반영 **[auto]**
- [ ] **글로우** ON → 후광 **[auto: 렌더]**
- [ ] **클릭 스프링**: 클릭 시 링이 살짝 줄었다 튕겨 복귀(squash→bounce) **[auto: 스프링 적분]** ← 실HW에서 눈으로
- [ ] 60Hz에서 모션이 매끄러운지(스프링·이동)

## D. 효과 (설정 토글별 ON/OFF)
- [ ] 좌/우클릭 ripple, 더블클릭 강조 **[auto]**
- [ ] 트레일(잔상) / 코멧 꼬리(드래그) **[auto: 게이팅]**
- [ ] 스크롤 방향 화살표(세로/가로) **[auto]**
- [ ] **흔들기로 찾기**: 커서 빠르게 흔들면 펄스 **[auto: 감지]** ← 실 마우스로
- [ ] **정지 펄스**: 커서 멈추면 idleTimeout 후 1회 펄스 **[auto: idle 감지]** ← 타이밍 체감
- [ ] 각 효과 설정 OFF 시 사라짐 **[auto: 게이팅 테스트]**

## E. 드래그 힌트
- [ ] 좌클릭 드래그 시 기준선(origin→current) / 속도 후광 **[auto]**
- [ ] **드래그 각도 라벨**(방향 화살표 + 각도) ON 시 표시 **[auto: 렌더]** ← 실 드래그로 각도 정확성

## F. 키스트로크
- [ ] 키 입력 표시 ON + Ctrl/Alt/Win 단축키 → 하단 카드에 표기 **[auto: KeyFormat]**
- [ ] 단순 타이핑·패스워드는 표시 안 됨(게이트) **[auto]**
- [ ] timeout 후 자동 숨김 **[auto]**
- [ ] **실 키보드 레이아웃**(한/영, 특수키)에서 글리프·문자 매핑 정확 ← US 외 레이아웃 주의(ToUnicodeEx 미사용)

## G. 그리기 모드 (★ 입력 캡처 토글 P1 — 라이브 미검증)
- [ ] **⌃⌥D**로 진입/이탈, 트레이 토글도 동작
- [ ] 진입 시 **클릭통과 OFF**(오버레이가 마우스 캡처) → 밑 앱 안 눌림 **[auto: ex-style 토글]** ← 실 드래그로
- [ ] 펜/직선/화살표/사각형/타원/형광펜/뱃지 7도구(모디파이어 매핑) **[auto: DrawingState]**
- [ ] **라이브 프리뷰**: 드래그 중 선이 실시간으로 보임 **[auto: 프레임 포함]** ← 체감
- [ ] `[` `]` 두께 조절(+영구화), `Ctrl+Z` undo, `ESC` 전체 지우고 이탈 **[auto]**
- [ ] 이탈 시 클릭통과 복원

## H. 라디얼 메뉴 (★ chord hold 라이브 미검증)
- [ ] **⌃⌥, 누르고 있기** → 0.15s 후 메뉴 표시(빠른 flick은 시각 없이 실행)
- [ ] 커서로 sector 선택, sub/subSub fan 라벨 표시 **[auto: 3단계 렌더]**
- [ ] **위로 = 스포트라이트(sector 0)** 자연스럽게 일치 **[auto: y-반전]** ← 실 커서로 방향 정합
- [ ] 떼면 선택 실행(설정/런타임 변경) **[auto: dispatch]**
- [ ] 라디얼 중 일반 클릭/효과 억제

## I. inspector 좌표
- [ ] **⌃⌥I**/트레이 토글 → 커서 옆 `x y` 좌표 라벨 **[auto: 렌더]**
- [ ] 좌표가 **전역 물리 픽셀**과 일치(멀티모니터에서 보조 모니터 좌표·음수 좌표 포함) ← §M와 연계

## J. 트레이
- [ ] 메뉴 항목 클릭으로 그리기/좌표 토글, **체크 표시가 현재 상태 반영** **[auto: MenuProvider]** ← 라이브
- [ ] **설정...** → 설정창 열림(단일 인스턴스, 두 번 클릭 시 앞으로)
- [ ] 후킹 분실(T2) 시 **풍선 알림** ("재설치되었습니다") ← §N에서 유발

## K. 설정 (라이브 + 영구화)
- [ ] 설정창 컨트롤 변경 → 오버레이 즉시 반영 **[auto: Changed→Apply]** ← 라이브
- [ ] 앱 재시작 후 설정 유지(%APPDATA%\Cluxo\settings.json) **[auto: 라운드트립]**
- [ ] 슬라이더 빠르게 움직여도 디바운스로 과도한 디스크쓰기 없음 **[auto: 디바운스]**
- [ ] settings.json 손상/삭제 시 기본값 fallback **[auto]**

## L. 자동 실행
- [ ] 설정창 "로그인 시 실행" ON → HKCU Run 등록, **재로그인 시 자동 시작** **[auto: 레지스트리]** ← 실 로그아웃/로그인
- [ ] OFF → 등록 해제

## M. 멀티모니터 · DPI (★★ 최우선 — VM이 가장 많이 가림)
- [ ] **모니터마다** 독립 오버레이, 효과가 올바른 모니터에 나타남 **[auto: per-monitor 필터]**
- [ ] **좌표 정합**: 링/효과가 실제 커서 위치에 정확히(밀림 없음). 특히 **150%/200% DPI**에서
- [ ] **혼합 DPI**: 주 150% + 보조 100% 등에서 각 모니터 정합(로컬 DIP 변환 §4)
- [ ] **음수 좌표** 모니터(주 모니터 왼쪽/위)에서 정합
- [ ] **핫플러그**: 모니터 연결/해제 시 오버레이 재구성, 크래시 없음 **[auto: MonitorsChanged 배선]** ← 실 케이블
- [ ] **DPI 변경**(설정에서 배율 변경/모니터 간 창 이동) 후 정합 유지
- [ ] 모니터별 다른 주사율에서 페이싱(주 모니터 vsync 기준 — 보조에서 티어링 허용 여부 판단)

## N. 입력 후킹 견고성 (★ T2 — 실부하 미검증)
- [ ] LL 마우스/키보드 후킹이 **실 사용 부하**(빠른 마우스·게임·고DPI 마우스)에서 끊김 없이
- [ ] 콜백 지연으로 OS가 후킹 제거하지 않는지(경량 콜백 설계) **[auto: 큐 분리]**
- [ ] **세션 전환/UAC 프롬프트(보안 데스크톱)** 후 후킹 복구 + 풍선 알림(T2)
- [ ] 잠금/절전→복귀 후 정상
- [ ] **관리자 권한 앱** 포그라운드 시 후킹/오버레이 동작(또는 한계 문서화)
- [ ] 핫키 충돌(다른 앱이 ⌃⌥D 점유) 시 동작/안내

## O. 화면 캡처 / 녹화 (★ 발표 안전 — 핵심 용도)
- [ ] **OBS / Zoom / Teams 화면공유**에 오버레이 효과가 **보임**(WDA_NONE 기본)
- [ ] 녹화 영상에 링/그리기/키스트로크가 캡처됨
- [ ] 발표 앱(PowerPoint 슬라이드쇼·Keynote) 전체화면 위에 오버레이 표시
- [ ] (발표앱 감지 동작은 코디네이터 TODO — 현재 미동작, 회귀 아님)

## P. 성능
- [ ] 유휴 시 CPU ~0%, 효과 활성 시 합리적(단일 모니터 한 자리 %)
- [ ] 멀티모니터에서 GPU/CPU 과부하 없는지
- [ ] **첫 실행 콜드 스타트**(WPF 합성 ~0.5s + 단일파일 압축해제) 수용 가능한지
- [ ] 장시간(수시간) 구동 시 메모리 누수 없음

## 알려진 한계 / 스톱갭 리스크 (QA 시 인지)
- **렌더 백엔드 = WPF 투명 윈도우 스톱갭**(Vortice/D2D 미사용). `AllowsTransparency`는 레이어드 합성이라
  60Hz 다중효과·다중모니터에서 성능 한계 가능 → 본 QA에서 성능 기준 미달이면 **D2D 백엔드 교체** 우선순위 상향.
- **키 문자 매핑**: US 레이아웃 기준 정적 테이블(ToUnicodeEx 미사용). 비-US 레이아웃 특수문자 주의.
- **라디얼 sub/subSub fan 라벨**만 렌더(아이콘·중앙 현재값 일부 미구현).
- **스포트라이트 dim·돋보기**는 v1.1(미구현, 회귀 아님).
- **모니터 안정 ID = szDevice**(재배열 시 변동 가능) → 신뢰 모니터/발표안전 기능 쓰면 DisplayConfig 기반으로 교체 필요.
